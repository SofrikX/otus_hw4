# Обработка ошибок и негативные сценарии

## 1. Авторизация и регистрация

| Ошибка | Поведение системы | UX-решение |
|---|---|---|
| Почта уже зарегистрирована | Регистрация не выполняется | Показать сообщение и предложить вход |
| Неверный пароль | Вход не выполняется | Показать ошибку под полем пароля |
| Слабый пароль | Регистрация блокируется | Показать требования к паролю |
| Нет интернета | Запрос не отправляется | Показать кнопку "Повторить" |

## 2. Профиль питомца

| Ошибка | Поведение системы | UX-решение |
|---|---|---|
| Не указано имя | Профиль не сохраняется | Подсветить поле имени |
| Некорректный возраст | Сохранение блокируется | Показать подсказку "Возраст от 0 до 30" |
| Фото не загрузилось | Профиль сохраняется без фото или загрузка повторяется | Кнопка "Повторить загрузку" |

## 3. Публикации

| Ошибка | Поведение системы | UX-решение |
|---|---|---|
| Нет фото | Публикация не создается | Сообщение "Добавьте хотя бы одно фото" |
| Файл больше 10 МБ | Файл не принимается | Предложить выбрать другой файл |
| Нет интернета | Публикация сохраняется как черновик | Сообщение "Будет отправлено позже" |
| Сервер недоступен | Отправка останавливается | Кнопка "Повторить" |

## 4. Лента

| Ошибка | Поведение системы | UX-решение |
|---|---|---|
| Нет публикаций | Показывается пустое состояние | Кнопка "Создать первый пост" |
| Лента не загрузилась | Показывается сохраненная версия | Кнопка "Обновить" |

## 5. Сообщения

| Ошибка | Поведение системы | UX-решение |
|---|---|---|
| Сообщение не отправлено | Статус "Не отправлено" | Кнопка повтора |
| Собеседник удалил аккаунт | Отправка блокируется | Текст "Пользователь недоступен" |

## 6. Прогулки

| Ошибка | Поведение системы | UX-решение |
|---|---|---|
| Геолокация запрещена | Прогулки рядом не определяются | Кнопка перехода в настройки |
| Нет прогулок рядом | Пустой список | Предложить создать прогулку |
| Прогулка отменена | Присоединение невозможно | Обновить список и показать сообщение |

## 7. Общие правила

1. Ошибка должна быть понятной для пользователя.
2. Если действие можно повторить, должна быть кнопка "Повторить".
3. Если данных нет, показывается пустое состояние.
4. При ошибке формы подсвечивается конкретное поле.
5. Пользователь не должен терять введенный текст при ошибке.
