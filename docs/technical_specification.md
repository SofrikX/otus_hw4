# Техническое задание — PetConnect

## 1. Назначение документа

Документ описывает требования к мобильному приложению PetConnect. Техническое задание подготовлено так, чтобы по нему можно было начинать разработку MVP на Flutter.

## 2. Описание проекта

PetConnect — мобильная социальная сеть для владельцев домашних животных. Приложение позволяет создавать профили питомцев, публиковать фотографии, взаимодействовать с публикациями, общаться в личных сообщениях и искать прогулки рядом.

### Правила архитектуры

1. Экран не должен напрямую обращаться к Firebase.
2. Вся работа с данными идет через слой репозиториев.
3. Бизнес-логика выносится из виджетов.
4. Повторяемые элементы интерфейса размещаются в `core/widgets`.
5. Ошибки приводятся к понятным пользовательским сообщениям.

## 6. Функциональные требования

### 6.1 Авторизация

Функции:

- регистрация;
- вход;
- выход;
- восстановление пароля.

Поля регистрации:

| Поле | Тип | Обязательное | Ограничения |
|---|---|---|---|
| Почта | строка | Да | Корректный формат |
| Пароль | строка | Да | Минимум 8 символов |
| Повтор пароля | строка | Да | Должен совпадать |

Бизнес-правила:

1. Нельзя зарегистрироваться с уже существующей почтой.
2. После успешного входа пользователь попадает на главный экран.
3. При выходе очищается локальная сессия.

### 6.2 Профиль питомца

Поля:

| Поле | Тип | Обязательное | Ограничения |
|---|---|---|---|
| Имя | строка | Да | До 50 символов |
| Вид животного | список | Да | Собака, кошка, другое |
| Порода | строка | Нет | До 80 символов |
| Возраст | число | Нет | От 0 до 30 |
| Описание | строка | Нет | До 500 символов |
| Фото | файл | Нет | JPG/PNG до 10 МБ |

Бизнес-правила:

1. У одного пользователя может быть несколько питомцев.
2. У питомца обязательно должно быть имя.
3. Если фото не загружено, используется стандартная иконка.

### 6.3 Лента публикаций

Состав публикации:

| Поле | Тип | Обязательное |
|---|---|---|
| Автор | пользователь | Да |
| Питомец | питомец | Да |
| Текст | строка | Нет |
| Изображения | список файлов | Да |
| Дата создания | дата | Да |
| Лайки | число | Да |
| Комментарии | список | Нет |

Бизнес-правила:

1. Публикация не может быть создана без изображения.
2. Максимум 5 изображений.
3. Размер изображения до 10 МБ.
4. Текст до 1000 символов.
5. Пользователь может удалить только свою публикацию.

### 6.4 Комментарии и лайки

Требования:

1. Пользователь может поставить лайк.
2. Повторное нажатие снимает лайк.
3. Пользователь может оставить комментарий.
4. Пустой комментарий отправить нельзя.
5. Комментарий до 500 символов.

### 6.5 Личные сообщения

Требования:

1. Пользователь может открыть чат.
2. Пользователь может отправить сообщение.
3. Пустое сообщение отправить нельзя.
4. При ошибке отправки отображается статус "Не отправлено".
5. Чаты сортируются по последнему сообщению.

### 6.6 Прогулки

Поля прогулки:

| Поле | Тип | Обязательное |
|---|---|---|
| Название | строка | Да |
| Место | строка | Да |
| Дата и время | дата | Да |
| Описание | строка | Нет |
| Организатор | пользователь | Да |
| Участники | список | Нет |

Бизнес-правила:

1. Нельзя создать прогулку в прошлом.
2. Организатор автоматически становится участником.
3. Пользователь может присоединиться к прогулке.
4. Пользователь может выйти из прогулки.
5. Отмененная прогулка недоступна для присоединения.

## 8. Требования к UI

1. Использовать Material 3.
2. Основной стиль MVP — яркая социальная сеть.
3. Основные экраны должны иметь нижнюю навигацию.
4. Для загрузки данных использовать индикаторы загрузки.
5. Для пустых состояний использовать поясняющий текст и кнопку действия.

Основные экраны:

| Экран | Назначение |
|---|---|
| Вход | Авторизация |
| Регистрация | Создание аккаунта |
| Главная лента | Просмотр публикаций |
| Создание публикации | Добавление фото и текста |
| Профиль питомца | Информация о питомце |
| Чаты | Список переписок |
| Прогулки | Поиск и создание прогулок |

## 10. Обработка ошибок

Подробно описана в файле `error_handling.md`.

Ключевые типы:

1. Ошибки валидации.
2. Сетевые ошибки.
3. Отсутствие данных.
4. Неавторизованный доступ.
5. Ошибки загрузки изображений.

7. Ошибки доступа к геолокации.

## 11. Критерии готовности MVP

MVP считается готовым, если:

1. Пользователь может зарегистрироваться и войти.
2. Пользователь может создать профиль питомца.
3. Пользователь может создать публикацию с фото.
4. Пользователь может просматривать ленту.
5. Пользователь может ставить лайки и писать комментарии.
6. Пользователь может отправлять сообщения.
7. Пользователь может создать прогулку.
8. Ошибки валидации отображаются пользователю.
9. Сетевые ошибки обработаны.
10. Все Must Have истории реализованы и протестированы.

## 13. Не входит в MVP

1. Платные подписки.
2. Реклама.
3. Видеозвонки.
4. Веб-версия.
5. Профили ветеринарных клиник.
