# User Story Mapping — PetConnect

## Роли

| Роль | Описание |
|---|---|
| Гость | Пользователь без учетной записи |
| Владелец питомца | Основной пользователь приложения |
| Организатор прогулки | Пользователь, который создает встречу |
| Участник прогулки | Пользователь, который ищет прогулку рядом |

## Основной пользовательский путь

1. Пользователь регистрируется.
2. Создает профиль питомца.
3. Смотрит ленту.
4. Публикует фото питомца.
5. Получает лайки и комментарии.
6. Пишет сообщение другому владельцу.
7. Находит прогулку рядом.

---

## Must Have

### US-1. Регистрация пользователя

Как гость, я хочу зарегистрироваться по электронной почте, чтобы получить доступ к приложению.

Приоритет: Must Have

```gherkin
Given пользователь находится на экране регистрации
When вводит корректную почту и пароль
Then система создает учетную запись и открывает главный экран
```

```gherkin
Given пользователь ввел почту, которая уже используется
When нажимает кнопку регистрации
Then система показывает сообщение "Эта почта уже зарегистрирована"
```

```gherkin
Given пользователь ввел пароль короче 8 символов
When нажимает кнопку регистрации
Then система показывает сообщение "Пароль должен содержать минимум 8 символов"
```

### US-2. Вход в приложение

Как пользователь, я хочу войти в приложение, чтобы пользоваться своим профилем и функциями.

Приоритет: Must Have

```gherkin
Given пользователь находится на экране входа
When вводит корректную почту и пароль
Then система открывает главный экран приложения
```

```gherkin
Given пользователь ввел неверный пароль
When нажимает кнопку входа
Then система показывает сообщение "Неверная почта или пароль"
```

### US-3. Создание профиля питомца

Как владелец питомца, я хочу создать профиль питомца, чтобы рассказывать о нем другим пользователям.

Приоритет: Must Have

```gherkin
Given пользователь авторизован
When заполняет имя, вид животного, возраст и описание
Then система сохраняет профиль питомца
```

```gherkin
Given пользователь не указал имя питомца
When нажимает кнопку сохранения
Then система подсвечивает поле имени и показывает ошибку
```

### US-4. Создание публикации

Как владелец питомца, я хочу публиковать фотографии питомца, чтобы делиться моментами с сообществом.

Приоритет: Must Have

```gherkin
Given пользователь авторизован и открыл экран создания публикации
When выбирает фото и вводит текст
Then публикация появляется в ленте
```

```gherkin
Given пользователь не выбрал фото
When нажимает кнопку публикации
Then система показывает сообщение "Добавьте хотя бы одно фото"
```

```gherkin
Given отсутствует интернет-соединение
When пользователь пытается опубликовать запись
Then система показывает сообщение о сетевой ошибке и кнопку повторной отправки
```

### US-5. Просмотр ленты публикаций

Как пользователь, я хочу просматривать ленту публикаций, чтобы видеть обновления питомцев.

Приоритет: Must Have

```gherkin
Given пользователь авторизован
When открывает главный экран
Then система показывает список последних публикаций
```

```gherkin
Given новых публикаций нет
When пользователь открывает ленту
Then система показывает экран без данных с предложением создать первую публикацию
```

### US-6. Лайки и комментарии

Как пользователь, я хочу ставить лайки и писать комментарии, чтобы взаимодействовать с публикациями.

Приоритет: Must Have

```gherkin
Given пользователь видит публикацию
When нажимает кнопку лайка
Then счетчик лайков увеличивается на 1
```

```gherkin
Given пользователь ввел текст комментария
When нажимает кнопку отправки
Then комментарий появляется под публикацией
```

### US-7. Личные сообщения

Как пользователь, я хочу отправлять личные сообщения, чтобы общаться с другими владельцами животных.

Приоритет: Must Have

```gherkin
Given пользователь открыл чат с другим пользователем
When вводит сообщение и нажимает отправить
Then сообщение отображается в переписке
```

```gherkin
Given сервер недоступен
When пользователь отправляет сообщение
Then система показывает статус "Не отправлено" и кнопку повтора
```

## Should Have

### US-8. Поиск прогулок рядом

Как владелец питомца, я хочу видеть прогулки рядом, чтобы присоединиться к другим владельцам.

Приоритет: Should Have

```gherkin
Given пользователь разрешил доступ к геолокации
When открывает экран прогулок
Then система показывает список ближайших прогулок
```

### US-9. Создание прогулки

Как организатор прогулки, я хочу создать встречу, чтобы пригласить других владельцев животных.

Приоритет: Should Have

```gherkin
Given пользователь открыл экран создания прогулки
When выбирает место, дату и время
Then система создает прогулку и показывает ее в списке
```

### US-10. Поиск пользователей

Как пользователь, я хочу искать других владельцев животных, чтобы находить знакомых и новых друзей.

Приоритет: Should Have

```gherkin
Given пользователь открыл поиск
When вводит имя пользователя
Then система показывает подходящие профили
```

---

## Nice To Have

### US-11. Истории питомцев

Как пользователь, я хочу публиковать короткие истории питомца, чтобы делиться моментами на 24 часа.

Приоритет: Nice To Have

```gherkin
Given пользователь выбрал фото или видео
When публикует историю
Then история отображается в верхней части ленты
```

### US-12. Уведомления

Как пользователь, я хочу получать уведомления о лайках, комментариях и сообщениях, чтобы не пропускать активность.

Приоритет: Nice To Have

```gherkin
Given пользователь получил новое сообщение
When приложение закрыто
Then система отправляет push-уведомление
```
