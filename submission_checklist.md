# Submission Checklist — PetConnect / OpenAI Codex

Этот файл — чек-лист для студента, а не инструкция для Codex.

Инструкции для Codex находятся в `AGENTS.md`. Этот файл нужен, чтобы не забыть шаги перед сдачей ДЗ.

## 1. Подготовка проекта

- [ ] Распаковать архив.
- [ ] Открыть папку проекта в OpenAI Codex.
- [ ] Убедиться, что в корне есть `AGENTS.md`.
- [ ] Попросить Codex прочитать `AGENTS.md` и `docs/documents_index.md`.
- [ ] Убедиться, что Codex понимает стек: Flutter, Dart, Riverpod, go_router, Material 3.

## 2. Проверка структуры

- [ ] Есть `pubspec.yaml`.
- [ ] Есть `README.md`.
- [ ] Есть `development_report.md`.
- [ ] Есть `prompts.md`.
- [ ] Есть `AGENTS.md`.
- [ ] Есть `docs/documents_index.md`.
- [ ] Есть папки `lib/`, `test/`, `docs/`, `web/`.

## 3. Проверка функций

- [ ] Работает лента публикаций питомцев.
- [ ] Работают лайки.
- [ ] Работает добавление комментария на уровне mock-состояния.
- [ ] Работает список питомцев.
- [ ] Работает экран профиля питомца.
- [ ] Работает список прогулок.
- [ ] Работает присоединение к прогулке.
- [ ] Отображается экран чатов.

## 4. Проверка тестов

- [ ] Запущен `flutter pub get`.
- [ ] Запущен `dart format .`.
- [ ] Запущен `flutter analyze`.
- [ ] Запущен `flutter test`.
- [ ] Минимум 3 теста проходят.

## 5. Проверка адаптивности

- [ ] Приложение проверено на desktop-ширине.
- [ ] Приложение проверено на мобильной ширине браузера или эмулятора.
- [ ] Сделан скриншот desktop-вида.
- [ ] Сделан скриншот мобильного вида.
- [ ] При наличии UI-проблем скриншот передан Codex для анализа.

## 6. Доказательства AI-agent workflow

- [ ] В `prompts.md` есть ключевые промпты.
- [ ] В `development_report.md` описано использование OpenAI Codex.
- [ ] Есть скриншот Codex с задачей анализа проекта.
- [ ] Есть скриншот Codex с задачей исправления ошибки или теста.
- [ ] Есть скриншот терминала с `flutter test`.

## 7. GitHub

- [ ] Создан GitHub-репозиторий.
- [ ] Код загружен в репозиторий.
- [ ] README виден на GitHub.
- [ ] В репозитории есть тесты.
- [ ] В репозитории есть `development_report.md`.
